# Публичный сайт с аккаунтами (готовый деплой)

Теперь это **полноценный сайт**:
- регистрация/вход
- прогресс сохраняется на сервере под вашим аккаунтом
- по ссылке может зайти **любой**

## Локальный запуск (проверить у себя)

1) Запустите `start-server.bat`
2) Откройте: `http://localhost:5173/`

## Публичная ссылка (Render, бесплатно)

Render умеет держать Node‑сервер и **постоянный диск**, чтобы база пользователей не пропадала.

1) Создайте аккаунт на Render
2) New → **Blueprint**
3) Подключите репозиторий с этим проектом (GitHub)
4) Render сам прочитает `render.yaml` и создаст сервис
5) Через 1–3 минуты получите публичный URL вида `https://productiv.onrender.com`

### Важно

- Прогресс хранится в `data/db.json` на диске Render.
- Если вы удалите сервис/диск — база пользователей пропадёт.

## Альтернатива: Docker

Если у вас есть любой VPS/хостинг под Docker:

```bash
docker build -t productiv .
docker run -p 5173:5173 -v productiv-data:/app/data productiv
```
